Cross-Site Request Forgery:

Lab: https://portswigger.net/web-security/csrf

  • Attacker can Forge HTTP Request of your Website —> Then, Trick User to Click/open (Malicious Script or page) link —> Do, Unintended Action (Change Email, PWD, Transfer Fund, Gain Full Control over User Account)
  • Allow Attacker Move Same Origin Policy: which prevent Different Website from interfering with each other: ![[cross-site request forgery.svg]]
  • Most IMPORTAINT:
  • CSRF Attack be Possible; 3 Condition Must Meet:
    • Relevant Action: Reason for attack the Victim (Gain Privileged User, Change User PWD or Email)
    • Cookie-based Session Handling: Session Cookie identify user —-> Any Other Mechanism (Tracking Session or Validate user Request) Won’t Work
    • No Unpredictable Request Parameters: All the Parameter Need to Stay the Same
    • Ex: Request:
Host: vulnerable-website.com
Content-Type: application/x-www-form-urlencoded
Content-Length: 30
Cookie: session=yvthwsztyeQkAPzeQ5gHgTvlyxHfsAfE

email=wiener@normal-user.com

  • Same-Site Cookie Restriction: Determines when website Should include cookies Request from other website

    • What is Site: ![[Same-site site.png]]http:// to https:// Treated as Cross-site by Most Browser

    • Different btw Site and Origin: ![[Site vs Origin.png]]Change in :PORT, Sub&Domain, & Scheme

    • Lax: Same-Site restriction mean: Browser will send Cookie in cross-site request; Only IF,

      • Request use GET method

      • Request from ==Top-Level Domain== by user (Clicking link)

      • LAB: Bypassing Same-Site via Lax GET method override:

        • login: “wiener” —> ==Repeater==: login & Update email

        • Respond of: POST /login Don’t have Any Same-Site restriction —> Browser Default use Lax restriction Level –Which mean–> Cookie: session= will be sent in Cross-Site GET request

        • POST /my-account/change-email HTTP/1.1 –> ==Burp –> Change request method== GET /my-account/change-email?email=t1%40mail.com HTTP/1.1 —> Only Allow POST request —> Add &_method=POST: GET /my-account/change-email?email=t1%40mail.com&_method=POST
